# 08 世界知识固化（Lore Canonicalization）

## 与世界 Agent 的关系

世界 Agent 持有两类知识，职责不同：

| 类型 | 性质 | 来源 | 举例 |
|------|------|------|------|
| 动态世界状态 | 事件驱动，持续变化 | 事件 Agent 广播 | 某地点当前状况、势力关系 |
| 静态世界知识（Lore） | 背景事实，相对稳定 | 作者预设 + 本模块固化 | NPC 的过去经历、地名由来、组织历史 |

**本模块负责过程，世界 Agent 负责存储。**
本模块是向世界 Agent Lore 层写入数据的唯一管道。

---

## 触发时机

每次 NPC 生成回复后，本模块对回复内容进行扫描，判断是否包含**新陈述的事实**。

不是每条回复都会触发写入——只有满足以下条件时才处理：
- NPC 陈述了一个关于自身、他人、或世界的具体事实
- 该事实在现有 Lore 中不存在（即为新生成内容）
- 该事实具有可复现性（未来可能被再次问及或引用）

闲聊、情绪表达、意见性陈述不触发固化。

---

## 事实分类

提取到的事实按影响范围分类，写入不同位置：

| 类型 | 举例 | 写入位置 |
|------|------|----------|
| NPC 个人事实 | "我在雷瓦霍尔上的学" | NPC 角色档案 + 世界 Agent Lore |
| 世界事实 | "这座城市建于两百年前" | 世界 Agent Lore |
| 关系事实 | "我认识警察局长" | 双方 NPC 档案 + 世界 Agent Lore |
| 组织/机构事实 | "工会在十年前分裂过" | 世界 Agent Lore |

NPC 个人事实同时写入 NPC 角色档案（本地缓存）和世界 Agent（单一事实权威）。
世界 Agent 是最终的事实权威，NPC 档案是检索加速层。

---

## 一致性验证

写入前必须检查新事实与现有 Lore 是否冲突：

```
新事实
    ↓
查询世界 Agent Lore（相关条目）
    ├── 无冲突 → 直接写入，标记来源为"AI 生成"
    ├── 有冲突
    │     ├── 作者预设优先 → 新事实丢弃，标记该 NPC"说错了"
    │     └── 与已固化 AI 事实冲突 → 先陈述的版本优先，新事实丢弃
    └── 相关但不冲突 → 写入，并建立关联索引
```

**首次陈述原则**：一旦某个事实被固化，后续的矛盾陈述视为 NPC 的记忆偏差或口误，不覆盖已有 Lore。这实际上增加了 NPC 的真实感。

---

## 事实权威层级

```
Level 1：作者预设（硬编码设定）       ← 不可覆盖，不可修改
Level 2：AI 生成已固化（正典）         ← 首次陈述原则，后续不可覆盖
Level 3：对话中生成、尚未固化          ← 本模块处理前的临时状态
```

查询时按层级优先返回。

---

## 传播机制

新事实写入后，需要通知可能受影响的其他 Agent：

**直接传播**：关系事实涉及的 NPC 双方都需要更新
```
NPC A 说"我认识 NPC B"
    → NPC B 的档案中新增：NPC A 声称认识你（待 NPC B 确认）
```

**按需传播**：世界事实不主动推送，其他 NPC 在下次被相关话题激活时，
从世界 Agent 检索到最新 Lore，自然获得一致性。

---

## 与其他模块的接口

- ← NPC 角色 Agent：每次生成回复后触发扫描
- → 世界 Agent Lore 层：写入固化的事实
- → 相关 NPC 角色档案：写入个人事实的本地缓存
- ← 仲裁层：查询时可请求 Lore 层数据（用于叙事可行性检查）
