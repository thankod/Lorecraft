# 06 记忆架构

## 核心设计原则

同一事件存在两种版本，分别由不同 Agent 持有：

| 版本 | 持有者 | 性质 | 用途 |
|------|--------|------|------|
| 客观版本 | 世界 Agent | 事实记录，不可扭曲 | 物理/空间可行性判断的依据 |
| 主观版本 | 各角色 Agent | 角色的个人解读，可包含事实性错误 | 角色行为决策、社会可行性判断的依据 |

**信息不对称是设计意图，不是 bug。**
NPC 可以对同一事件有错误的记忆，并基于此产生自洽但客观上"错误"的行动。

---

## 主观版本的生成机制

事件 Agent 生成客观版本后，向所有相关 Agent 广播 Tier 1+2。
各角色 Agent **收到广播后，自行生成主观版本**，而非由事件 Agent 统一生成。

```
事件广播（Tier 1+2 客观版本）
    ↓ 各角色 Agent 独立接收
角色 Agent 处理：
    基于以下因素对事件进行主观解读：
    ├── 与涉事角色的历史关系
    ├── 当前情绪状态
    ├── 已有的先入之见与偏见
    └── 此刻掌握的信息（可能不完整）
    ↓
生成主观版本（Tier 2 主观摘要）
写入角色自己的长期记忆
```

主观版本可能与客观版本存在以下偏差：
- **信息缺失**：角色当时不知道某个关键背景
- **意图误读**：将中性行为解读为敌意，或反之
- **情绪扭曲**：强烈情绪导致细节失真
- **主动遗忘**：对不愿面对的事实产生认知屏蔽（可选机制，待定）

## 记忆结构

### 世界 Agent 记忆（客观）

存储所有事件的完整客观版本（Tier 1-4），是系统的唯一事实权威。

```
世界记忆
├── 事件索引（Tier 1 + Tier 2）
│     用于快速检索，始终驻留
└── 事件全文（Tier 3 + Tier 4）
      按需加载，懒加载
```

### 角色 Agent 记忆（主观）

```
角色记忆
├── 近期缓冲（短期）
│     最近 N 个涉及自己的事件的主观摘要
│     直接注入 Agent 决策上下文
│     N 的大小根据角色重要性分级（主角 > 主要 NPC > 次要 NPC）
│
└── 长期记忆
      重要事件的主观摘要，持久化存储
      通过语义检索按需召回
      召回后可进一步请求对应客观事件的 Tier 3+4（如需对照）
```

---

## 检索机制

采用 **RAG 结构**：

```
检索阶段：Tier 2 摘要作为向量索引
    ↓ 语义相似度匹配
召回阶段：按需加载 Tier 3+4 原文
    ↓ 注入 Agent 上下文
```

仲裁层查询时：
- 查**世界 Agent** → 获取客观事件 Tier 1+2，用于物理/叙事可行性判断
- 查**角色 Agent 记忆** → 获取主观摘要，用于社会/关系可行性判断

---

## 记忆的遗忘与衰减

- **近期缓冲**：滑动窗口，旧事件自动移出缓冲区，转入长期记忆或丢弃
- **长期记忆**：重要性标记决定保留优先级，次要事件在超出容量时可被压缩或丢弃
- **客观版本不衰减**：世界 Agent 的记录是永久的，角色记忆的衰减不影响客观事实

---

## 信息传播

NPC 之间可以互相"讲述"事件，信息通过对话传播：

```
NPC A 的主观记忆
    ↓ 通过对话事件传递给 NPC B
NPC B 基于 A 的叙述 + 自身偏见，生成二手主观版本
```

信息在传播中会进一步失真，产生谣言、误解、偏见的自然涌现。
