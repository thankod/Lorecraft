# 05 叙事轨道

## 设计意图

叙事轨道的目的是**防止玩家长期游离于主线之外**，而不是强制特定剧情结果。

- 不要求特定人物存活
- 不要求特定物品被获取
- 不强制特定事件发生
- 只施加**叙事重力**，让世界保持对主线的引力

修正是渐进的、世界内部的，玩家感受到的是"世界在动"，而不是"系统在推我"。

---

## 叙事轨道 Agent

专职 Agent，持续监控事件流，自主裁决是否需要干预以及干预级别。

### 输入
- 事件流（Tier 1+2，持续接收）
- 主线叙事结构（游戏作者预设的阶段与方向）
- 当前偏移评估（内容偏离程度 + 持续时长）

### 判断机制
Agent 不使用硬阈值，而是综合评估：
- 最近 N 个事件中，有多少与主线方向相关？
- 偏离已持续多久？
- 当前玩家行为是"迂回接近主线"还是"真正游离"？（防止打断有意义的探索）
- 上一次干预是什么时候？效果如何？

根据评估，Agent 自主选择不干预或启动某一级干预。

---

## 三级干预机制

### 第一级：内心声音提醒
**触发条件**：轻度偏离，持续时间较短

通过反思系统的第一层认知（如"规划"或"直觉"）向玩家角色注入一个内心声音，以角色自身的焦虑或直觉形式出现。

```
"规划"：你还记得你来这里是为了什么吗？
"直觉"：有什么事情感觉不太对——好像有什么更紧迫的事情在等着。
```

不打断当前交互，在自然的叙事间隙插入。

### 第二级：当前 NPC 话题渗透
**触发条件**：中度偏离，或第一级未产生效果

叙事轨道 Agent 向**当前正在交互的 NPC Agent** 注入上下文提示，使该 NPC 在对话中自然提及与主线相关的信息。

**关键原则**：注入不能破坏 NPC 的角色一致性。
只有当该 NPC 在逻辑上"有可能知道或关心"相关信息时，才使用此 NPC 进行渗透。
否则跳过此级，等待更合适的 NPC 进入交互。

**注入时机**：方案 B——异步并行，下一轮生效。
叙事轨道 Agent 与当前对话并行运行，不阻塞主流程。
决定注入后写入该 NPC 的待注入队列，在玩家下一条消息到来时附加进 NPC 上下文。

**注入形式**：信息层而非指令层。
```
❌ 指令层（破坏角色一致性）：
"在这段对话中你应该提到 [主线事件]。"

✅ 信息层（保持角色自主性）：
"你最近隐约听说了 [相关传言]，这让你有些不安。
 如果对话走向合适，你可能会提起——但你不会强行插入。"
```
NPC 自行决定要不要说、怎么说。叙事轨道 Agent 只给了它一个"有话可说"的理由。

### 第三级：世界主动施压
**触发条件**：严重偏离，或前两级均未产生效果

叙事轨道 Agent 触发一个或多个世界级干预，强度递增：

**方案 A：相关 NPC 主动寻找玩家**

与主线直接相关的 NPC 产生"寻找玩家"的自主目标，进入正常的 NPC 行动流程（经过仲裁层），最终在场景中出现并制造叙事压力。

**方案 B：外围 NPC 施加间接压力**
与主线间接相关的 NPC（如雇主、盟友、受影响的普通人）通过对话、态度变化、或事件向玩家施加压力，使"继续拖延"产生感知代价。

**方案 C：世界状态自然推进**
主线相关的世界事件在玩家缺席的情况下自然发展，产生玩家无法忽视的后果——不是惩罚玩家，而是让时间的流逝有重量。

第三级可组合使用，由叙事轨道 Agent 根据情境选择最自然的方案。

---

## 干预的克制性原则

- 相邻两次干预之间必须有足够间隔，避免玩家感到被催促
- 同一场对话中不同时触发一级和二级
- 三级干预不应在玩家刚刚开始一段新对话时触发
- 任何干预都不能在叙事层面"解释自己"——不存在"你应该去做主线任务"这样的表达

---

## 与其他模块的接口

- ← 事件 Agent：持续接收事件流 Tier 1+2
- → 反思系统：注入第一级内心声音
- → NPC 角色 Agent：注入第二级上下文提示
- → 事件 Agent：触发第三级世界事件（走正常事件生成流程）
