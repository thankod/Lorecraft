# 角色 Agent

适用于：玩家控制角色 + 所有 NPC

---

## 核心职责

每个角色 Agent 维护该角色的**主观视角**：
- 这个角色知道什么
- 这个角色相信什么（可能与事实不符）
- 这个角色当前的情绪与动机状态
- 这个角色对其他角色的关系与态度

---

## NPC 层级系统

NPC 按叙事重要性和交互深度分为三层，层级决定 Agent 能力与资源消耗：

### Tier A：主要 NPC
完整 Agent，全部能力开启。
```
完整长期记忆（RAG 检索）
主观版本生成（收到事件广播后独立解读）
目标队列（驱动自主行动）
关系图谱（完整）
简化版第一层反思系统（可选）
```
适用：驱动剧情的角色、反复出现的关键人物。

### Tier B：功能性 NPC
简化 Agent，保留核心交互能力，去除高消耗功能。
```
短期记忆（仅保留近期 N 次与玩家的交互摘要）
固定人格模板（无动态目标队列）
无主动主观版本生成（接收广播但不独立解读）
关系图谱（简化，仅记录对玩家的态度）
```
适用：有名字的店主、守卫、有固定职能的角色。

### Tier C：背景 NPC
无状态，每次交互从人格模板重新实例化，不保留记忆。
```
人格模板（类型：路人、小贩、醉汉……）
无记忆、无关系图谱、无目标
```
适用：人群、一次性路人、环境填充角色。
Tier C 的发言内容不需要跨会话一致——他们在世界里本就是可替换的。

---

## 层级动态升降

层级不是固定的，随玩家行为动态调整：

**升级（Tier C → Tier B）**
```
触发条件：与同一 Tier C NPC 交互超过 N 次
处理：
  1. 为该 NPC 创建记忆档案
  2. 将过去几次交互摘要作为初始记忆写入
  3. 从此按 Tier B 处理
```

**降级（Tier B 轻量化）**
```
触发条件：Tier B NPC 长期无交互
处理：
  1. 将记忆压缩为单条摘要
  2. 进入轻量模式（保留摘要，不再主动维护）
  3. 再次被交互时恢复正常 Tier B 处理
```

Tier A 不降级。

## Agent 内部结构（Tier A / 玩家角色）

```
角色 Agent
├── 身份与背景（静态，游戏开始时设定）
│     角色基础信息、世界观、核心动机
├── 当前状态（动态，随事件更新）
│     情绪状态、位置、手头资源
├── 关系图谱（动态）
│     对其他角色的态度、信任度、历史记录
│     采用语义描述而非纯数值，保留情感质感
│     （"警惕但好奇" 比 trust=0.6 更能指导 NPC 行为生成）
├── 记忆（动态，详见 06 记忆架构）
│     长期记忆（RAG）+ 近期事件缓冲
└── 目标队列（动态，NPC 专属）
      当前主动目标 + 优先级
      目标由 Agent 基于自身状态和世界变化自主生成
      目标达成或失效时从队列移除
```

## 玩家角色 vs NPC 的差异

| 维度 | 玩家角色 | Tier A NPC | Tier B NPC | Tier C NPC |
|------|----------|------------|------------|------------|
| 意图来源 | 玩家输入 | Agent 自主 | Agent 自主（有限） | 无 |
| 反思系统 | 完整双层 | 简化第一层 | 无 | 无 |
| 目标队列 | 不适用 | 有 | 无 | 无 |
| 长期记忆 | 完整 | 完整 | 短期摘要 | 无 |
| 主观版本生成 | 是 | 是 | 否 | 否 |
| 事件广播接收 | 全量相关 | 按权重+关系 | 仅同场景 | 不接收 |

---

## NPC 自主行动

Tier A / B NPC 会在以下时机产生自主意图：
- 接收到影响自己的事件广播
- 目标队列中有未完成目标（仅 Tier A）
- 世界状态发生影响自己利益的变化

NPC 的自主意图同样经过仲裁层，确保世界逻辑一致性。
