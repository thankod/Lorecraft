# 世界 Agent

## 概述

世界 Agent 是系统的**唯一客观事实权威**，持有两类相互独立的知识：

| 层 | 名称 | 性质 | 写入来源 |
|----|------|------|----------|
| 动态层 | 世界状态 | 事件驱动，持续变化 | 事件 Agent 广播 |
| 静态层 | 世界知识（Lore） | 背景事实，相对稳定 | 作者预设 + 08 世界知识固化模块 |

两层分开存储，分开检索，互不污染。

---

## 动态层：世界状态

追踪当前世界中可变的客观事实：

```
地点状态
  当前状态（可进入性、当前人员）
  变更因果链（每次状态变更记录：变更前状态 → 变更原因 → 变更后状态）
  查询时同时返回当前状态与因果链，帮助玩家回忆上下文

时间
  游戏内时间、已过回合数

势力/关系
  各派系当前状态、相互关系、资源
  同样维护变更因果链

角色位置
  各 NPC 当前所在地点（粗粒度）
```

**更新来源**：只由事件 Agent 广播触发，确保所有状态变更有事件记录可追溯。

**查询用途**：仲裁层第一、二层检查（物理/空间可行性）。

### 惰性求值原则

世界不在后台持续自主演化。未被玩家观测的区域保持"冻结"状态。

```
玩家进入新区域 / 与某 NPC 发生交互
    ↓
世界 Agent 检查该区域/NPC 的上次观测时间戳
    ↓
基于以下信息推算"这段时间应该发生了什么"：
  - 上次观测时的状态
  - 经过的游戏内时间
  - 该区域 NPC 的目标与动机
  - 期间发生的全局重大事件（如有）
    ↓
生成补算事件，更新至当前状态
    ↓
玩家正式进入/交互，看到的是已推算完毕的世界
```

补算结果作为正式事件写入历史记录，保持叙事一致性。
补算的深度根据 NPC 层级决定：Tier A 做完整推算，Tier B 做简化推算，Tier C 直接重置为默认状态。

---

## 静态层：世界知识（Lore）

存储关于世界的背景性事实，按权威层级组织：

```
Level 1（作者预设）
  游戏作者在开发时写入的硬设定，不可被 AI 覆盖

Level 2（AI 生成已固化）
  对话中即兴生成、经 08 模块验证后写入的正典
  遵循首次陈述原则，写入后不可被后续矛盾陈述覆盖
```

**查询用途**：
- 仲裁层叙事可行性检查
- NPC 角色档案的上游权威（NPC 档案是本层的本地缓存）
- 08 模块写入前的一致性验证依据

---

## 与其他模块的接口

- ← 事件 Agent：动态层状态更新
- ← 08 世界知识固化：Lore 层写入
- → 仲裁层：响应客观状态查询（只读）
- → NPC 角色 Agent：响应 Lore 检索请求（只读）
