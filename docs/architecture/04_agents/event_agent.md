# 事件 Agent

## 概述

事件 Agent 是系统状态变更的执行者与记录者。
它接收通过仲裁的动作意图，生成事件记录，并广播给相关 Agent。

"事件"是整个系统的核心数据结构——所有 Agent 的状态更新、记忆写入、轨道检测都以事件为单位流转。

---

## 事件数据结构

事件不做原子化拆解（会遗漏语义信息），以层级结构存储，支持懒加载。

```
Event
├── [Tier 1] 元数据（始终加载）
│     title       事件标题，简短标识符
│     timestamp   游戏内时间戳
│     location    发生地点
│     participants 涉及角色列表
│     tags        类型标签（对话/冲突/发现/关系变化...）
│
├── [Tier 2] 摘要（按需加载）
│     summary     1-3 句话的事件概述
│                 用于：轨道偏移检测、记忆检索索引、快速推理
│
├── [Tier 3] 上下文介绍（按需加载）
│     context     理解此事件所需的背景
│                 包含：此前相关事件、人物关系现状、场景状态
│                 在事件生成时由 AI 自动填写，确保事件可脱离主流程被理解
│
└── [Tier 4] 事件原文（按需加载）
      narrative   游戏叙事文本的原始内容
                  是信息的最高保真来源，也是角色 Agent 做深度反应的依据
```

---

## 懒加载策略

不同消费者按需请求不同层级：

| 消费者 | 所需层级 | 原因 |
|--------|----------|------|
| 叙事轨道偏移检测 | Tier 1 + 2 | 只需判断事件方向 |
| 反思系统信号 B 提取 | Tier 1 + 2 | 判断选择类型即可 |
| 角色 Agent 快速状态更新 | Tier 1 + 2 | 知道发生了什么 |
| 角色 Agent 深度情感反应 | Tier 3 + 4 | 需要完整叙事 |
| 记忆检索（索引阶段） | Tier 1 + 2 | 相关性判断 |
| 记忆检索（召回阶段） | Tier 3 + 4 | 实际使用内容 |
| 仲裁层历史查询 | Tier 1 + 2 | 判断前置条件是否满足 |

## 事件生成流程

```
仲裁层通过动作
    ↓
事件 Agent 接收意图 + 当前世界状态 + 相关角色状态
    ↓
AI 生成事件各层级内容
    ├── Tier 3（上下文）在生成时由 AI 主动填写
    │     确保事件在未来被孤立读取时仍可被理解
    └── Tier 4（原文）即叙事文本，同时也是呈现给玩家的内容
    ↓
事件写入持久化存储
    ↓
广播 Tier 1 元数据给所有相关 Agent
    └── 各 Agent 按需请求更深层级
```

---

## 负面后果权重

当事件携带来自反思系统的"强行推进"标记时，
事件 Agent 在生成叙事时会相应调整后果的严重程度。

```
标记级别：
  无标记           → 正常后果
  一次坚持         → 轻度负面后果（NPC 态度轻微变化）
  明确确认坚持     → 显著负面后果（关系损伤、机会关闭等）
```

---

## 事件广播机制

广播不是全量推送，由**事件权重 × NPC 层级 × 关系/距离**共同决定：

```
私密事件（仅 participants）
  只广播给直接参与者，不考虑层级

普通事件（低权重）
  Tier A NPC：直接相关（participants 或强关系）才接收
  Tier B NPC：同一场景内才接收
  Tier C NPC：不接收

重大事件（高权重）
  Tier A NPC：无论身在何处都接收 Tier 1+2
  Tier B NPC：同一地区接收 Tier 1
  Tier C NPC：不接收广播，但可通过叙事感知氛围变化
```

### 二次传播（信息扩散）

事件发生后，信息在世界中随时间扩散，由世界 Agent 定期触发：

```
初次广播（事件发生时）
  → 直接参与者 + 在场 NPC

二次广播（时间流逝后）
  → 通过关系网络扩散
  → 扩散内容为 Tier 2 摘要（原文不传播）
  → 各 NPC 接收后经主观过滤，自然产生信息失真
```

二次传播的失真不需要专门设计——摘要经过各 NPC 主观记忆系统处理后，谣言和误解自然涌现。
