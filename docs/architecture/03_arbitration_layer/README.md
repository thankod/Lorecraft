# 03 仲裁层

## 概述

仲裁层是引擎的**可行性裁判**，负责判断一个原子动作在当前状态下能否发生。
它通过**单次 LLM 调用**完成所有维度的评估，输出综合检查报告。
不通过时，由 LLM 直接生成叙事内拒绝文本——玩家永远不会看到系统语言。

---

## 五维评估

仲裁层在一次 LLM 调用中评估以下五个维度：

### 第一维：信息完整性
角色当前是否拥有执行此动作所需的信息？

- 角色是否知道目标的位置？
- 角色是否认识目标人物？
- 角色是否了解执行动作所需的前提知识？

> 关键原则：**玩家知道的不等于角色知道的。**

### 第二维：空间与状态可行性
物理/空间层面是否可行？

- 角色当前位置与目标位置之间是否存在可行路径？
- 是否有物理阻断（锁着的门、敌对区域、时间不对）？
- 目标对象当前的状态是否允许此交互（对方在场吗？清醒吗？）？

### 第三维：社会与关系可行性
社会情境层面是否可行？

- 当前关系状态是否允许此交互（陌生人 vs 熟人）？
- 场合是否合适（公开场合 vs 私下）？
- 是否有 NPC 会物理/社会性阻断此行为？

> **信息源说明**：第三维查询的是角色 Agent 的**主观记忆**，而非世界客观状态。
> NPC 的社会行为基于他们"以为发生了什么"，可能与事实不符。
> 这是设计意图——信息不对称本身是可玩内容。

### 第四维：叙事可行性
当前剧情阶段，此动作是否有意义？

- 所需的叙事前置条件是否已满足？
- 此动作是否会产生逻辑上的悖论（对方还不知道某件事，你却要揭穿他）？

### 第五维：叙事轨道检查
此动作是否会导致叙事严重偏轨？

- 是否会直接跳过多个关键节点？
- 是否触发轨道修正阈值？（详见 [05 叙事轨道](../05_narrative_rail/README.md)）

> **第五维永远不会导致拒绝**——它只标记 `drift_flag`，由异步的叙事轨道 Agent 决定干预方式。

---

## 检查结果处理

### 通过
动作传入事件 Agent 生成对应事件。
若携带"强行推进"标记（来自反思系统坚持机制），事件 Agent 在生成时附加负面后果权重。

### 不通过
LLM 在同一次调用中直接给出叙事内拒绝文本。
**任何情况下，都不向玩家暴露"检查失败"等系统语言。**

---

## 叙事内拒绝

拒绝必须发生在世界内部。LLM 根据失败的维度和上下文，自由选择合适的拒绝方式，例如：

### 叙事内消化
通过角色的认知或内心声音体现障碍，不出现系统提示。

> "你意识到你甚至不知道他在哪里。城里有人提到过他的名字，仅此而已。"

### 部分执行 + 自然中断
能执行的部分正常执行，不能执行的通过世界反应自然阻断。

> "你向城北走去——那是你仅有的线索。走到一半，你停下来。你准备好了吗？"

### 意图重新解读
对语义模糊的意图，不拒绝，而是呈现可能的路径并说明当前条件不足。

> "让他倒下——但你还没有筹码。说服需要他信任你，而他现在只知道你是个陌生人。"

### 代价式执行（仅用于坚持情形）
玩家明确坚持时，不阻止，但执行附带真实代价。

> "你冲进了他的办公室。结果是灾难性的，但这是你选择的。"

---

## LLM 输出格式

```json
{
  "passed": false,
  "checks": [
    { "dimension": "information_completeness", "passed": true, "reason": null },
    { "dimension": "physical_spatial", "passed": false, "reason": "市长办公室的门上着锁" },
    { "dimension": "social_relationship", "passed": true, "reason": null },
    { "dimension": "narrative_feasibility", "passed": true, "reason": null },
    { "dimension": "narrative_drift", "passed": true, "reason": null }
  ],
  "drift_flag": false,
  "rejection_narrative": "市长办公室的门紧锁着，你没有钥匙无法进入。"
}
```
