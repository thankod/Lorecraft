# 03 仲裁层

## 概述

仲裁层是引擎的**可行性裁判**，负责判断一个原子动作在当前状态下能否发生。
它是纯后端逻辑，其结果由反思系统或事件 Agent 翻译为叙事语言呈现给玩家。

---

## 五层检查

原子动作按顺序通过以下检查，任意一层不通过则进入拒绝流程：

### 第一层：信息完整性
角色当前是否拥有执行此动作所需的信息？

- 角色是否知道目标的位置？
- 角色是否认识目标人物？
- 角色是否了解执行动作所需的前提知识？

> 关键原则：**玩家知道的不等于角色知道的。**

### 第二层：空间与状态可行性
物理/空间层面是否可行？

- 角色当前位置与目标位置之间是否存在可行路径？
- 是否有物理阻断（锁着的门、敌对区域、时间不对）？
- 目标对象当前的状态是否允许此交互（对方在场吗？清醒吗？）？

### 第三层：社会与关系可行性
社会情境层面是否可行？

- 当前关系状态是否允许此交互（陌生人 vs 熟人）？
- 场合是否合适（公开场合 vs 私下）？
- 是否有 NPC 会物理/社会性阻断此行为？

> **信息源说明**：第三层查询的是角色 Agent 的**主观记忆**，而非世界客观状态。
> NPC 的社会行为基于他们"以为发生了什么"，可能与事实不符。
> 这是设计意图——信息不对称本身是可玩内容。

### 第四层：叙事可行性
当前剧情阶段，此动作是否有意义？

- 所需的叙事前置条件是否已满足？
- 此动作是否会产生逻辑上的悖论（对方还不知道某件事，你却要揭穿他）？

### 第五层：叙事轨道检查
此动作是否会导致叙事严重偏轨？

- 是否会直接跳过多个关键节点？
- 是否触发轨道修正阈值？（详见 [05 叙事轨道](../05_narrative_rail/README.md)）

---

## 检查结果处理

### 通过
动作传入事件 Agent 生成对应事件。
若携带"强行推进"标记（来自反思系统坚持机制），事件 Agent 在生成时附加负面后果权重。

### 不通过
根据失败层级，采用不同的叙事拒绝策略（见下节）。
**任何情况下，都不向玩家暴露"检查失败"等系统语言。**

---

## 叙事内拒绝策略

拒绝必须发生在世界内部，以下四种策略可组合使用：

### 策略 A：叙事内消化（默认首选）
通过角色的认知或内心声音体现障碍，不出现系统提示。

> "你意识到你甚至不知道他在哪里。城里有人提到过他的名字，仅此而已。"

### 策略 B：部分执行 + 自然中断
能执行的部分正常执行，不能执行的通过世界反应自然阻断。

> "你向城北走去——那是你仅有的线索。走到一半，你停下来。你准备好了吗？"

### 策略 C：意图重新解读
对语义模糊的意图，不拒绝，而是呈现可能的路径并说明当前条件不足。

> "让他倒下——但你还没有筹码。说服需要他信任你，而他现在只知道你是个陌生人。"

### 策略 D：代价式执行（仅用于坚持情形）
玩家明确坚持时，不阻止，但执行附带真实代价。

> "你冲进了他的办公室。结果是灾难性的，但这是你选择的。"

---

## 拒绝的分级

不同层级的检查失败，拒绝的"硬度"不同：

| 失败层 | 拒绝性质 | 处理方式 |
|--------|----------|----------|
| 信息完整性 | 软性，可通过行动解决 | 引导玩家收集信息 |
| 空间/状态 | 软性，有时可绕过 | 部分执行或提示替代路径 |
| 社会关系 | 中性，可通过关系建立改变 | 叙事内体现社会阻力 |
| 叙事可行性 | 中性到硬性 | 叙事内逻辑阻断 |
| 叙事轨道 | 硬性 | 世界事件主动介入 |
