# 07 Agent 通信

## 核心设计原则

**叙事轨道 Agent 是系统中唯一不在关键路径上的 Agent。**
其余所有 Agent 串行处理一次玩家输入；叙事轨道 Agent 始终异步并行运行，观察事件流，通过队列间接影响世界，从不阻塞任何人等它。

---

## 完整调用链

### 一次玩家输入的处理流程

```
玩家输入（自由文本）
        │
        ▼
┌─────────────────┐
│   01 输入层      │  意图解析 → 原子动作序列
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  02 反思系统     │  评估意图
│                 │  ├─ 合理 → 静默通过
│                 │  ├─ 存疑 → 生成内心声音文本，附加在输出队列
│                 │  └─ 拦截 → 生成内心声音，等待玩家确认是否坚持
└────────┬────────┘
         │ （通过或坚持后继续，携带可选的"强行推进"标记）
         ▼
┌─────────────────┐
│  03 仲裁层       │  五层可行性检查
│                 │  ├─ 通过 → 继续
│                 │  └─ 不通过 → 生成叙事拒绝文本，流程终止
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  04 事件 Agent  │  生成事件（Tier 1-4）
│                 │  写入持久化存储
│                 │  广播 Tier 1 元数据
└────────┬────────┘
         │ 广播
    ┌────┴─────────────────┐
    ▼                      ▼
┌──────────┐        ┌──────────────┐
│ 世界 Agent│        │ 相关角色 Agent│  （可并行）
│ 更新客观  │        │ 生成主观版本  │
│ 世界状态  │        │ 写入角色记忆  │
└──────────┘        └──────────────┘
```

### 并行运行（全程）
```
┌──────────────────────┐
│   05 叙事轨道 Agent   │  监控事件流
│                      │  ├─ 无需干预 → 静默
│                      │  ├─ 一级干预 → 写入反思系统注入队列
│                      │  ├─ 二级干预 → 写入目标 NPC 的待注入队列
│                      │  └─ 三级干预 → 向事件 Agent 提交 NPC 自主行动请求
└──────────────────────┘
```

---

## 各 Agent 的通信方式

| 通信方向 | 方式 | 说明 |
|----------|------|------|
| 输入层 → 反思系统 | 同步调用 | 传入原子动作序列 + 语气信号 |
| 反思系统 → 仲裁层 | 同步调用 | 传入原子序列 + 可选坚持标记 |
| 仲裁层 → 世界 Agent | 同步查询 | 查客观状态，只读 |
| 仲裁层 → 角色 Agent | 同步查询 | 查主观记忆，只读 |
| 仲裁层 → 事件 Agent | 同步调用 | 传入通过的动作 |
| 事件 Agent → 世界/角色 Agent | 异步广播 | 广播 Tier 1，各自按需拉取更深层级 |
| 叙事轨道 Agent → 反思系统 | 写入队列 | 一级干预，下次自然间隙插入 |
| 叙事轨道 Agent → NPC Agent | 写入队列 | 二级干预，下轮对话生效 |
| 叙事轨道 Agent → 事件 Agent | 提交请求 | 三级干预，走正常事件流程 |

---

## NPC 自主行动的流程

NPC 不只被动响应，也会主动产生意图：

```
NPC 角色 Agent 收到事件广播
        │ 或：目标队列中有未完成目标
        │ 或：叙事轨道 Agent 提交了行动请求
        ▼
NPC Agent 生成自主意图
        ▼
进入仲裁层（与玩家行为使用同一套检查）
        ▼
事件 Agent 生成事件，广播更新
```

NPC 的自主行动与玩家行动使用**完全相同的仲裁和事件流程**，保证世界逻辑一致性。

---

## 待注入队列

叙事轨道 Agent 通过两个队列间接影响世界，不直接修改任何 Agent 的运行状态：

**反思系统注入队列**
```
[{ voice: "规划", content: "...", priority: low, expiry: 3轮 }]
```
在下一个自然叙事间隙（如对话结束、场景转换）消费。

**NPC 待注入队列**
```
npc_id → [{ context: "你听说了...", condition: "合适时机", expiry: 5轮 }]
```
在该 NPC 下一轮生成回复时附加进其上下文。过期未消费则丢弃。
