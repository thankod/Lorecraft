# AI-Driven CRPG 框架架构总览

> 目标：构建一个由 AI 驱动、高自由度的对话式 CRPG 引擎框架。
> 参照：极乐迪斯科式叙事风格，无战斗，纯对话与选择驱动。
> 原则：一切语义相关的处理均由 AI 负责。

---

## 设计哲学

- **玩家输入是意图，不是指令** — 引擎负责解释、过滤、仲裁，而非直接执行
- **系统边界对玩家不可见** — 所有限制通过叙事内部消化，不暴露技术感
- **行为塑造人格，人格影响行为** — 玩家的选择模式动态改变角色的内在声音
- **信息不对称是设计意图** — NPC 可以持有错误的主观记忆并基于此行动
- **世界只向前运动** — 事件不可撤销，时间流逝有重量
- **惰性求值** — 未被观测的区域冻结，观测时按需推算
- **LLM 负责所有语义判断** — 规则只定义结构，判断交给模型

## 游戏生命周期

```
① 初始化（一次性）
   初始化 Agent 凭空生成：世界设定、叙事结构、NPC 档案、起始事件
       ↓ 分发
   世界 Agent（Lore 层）/ 各角色 Agent / 叙事轨道 Agent / 事件 Agent

② 运行时（循环）
   见下方数据流

③ 存档
   创世文档持久化，支持多次读取同一世界游玩
```

---

## 完整数据流

```
玩家自由文本输入
        ↓
[01 输入层]
  语义解析 → 意图提取 → 原子动作拆解 → 语义消歧
  同时提取语气信号（→ 反思系统信号 A）
        ↓
[02 反思系统]
  第一层（固定认知）+ 第二层（浮现特质，双信号权重驱动）
  ├─ 合理 → 静默通过
  ├─ 存疑 → 内心声音发言（辩论），玩家继续
  └─ 拦截 → 内心声音警告，等待玩家确认是否坚持
        ↓ （通过，可选携带"强行推进"标记）
[03 仲裁层]
  ① 信息完整性（角色知道吗？）      → 查角色主观记忆
  ② 空间/状态可行性                 → 查世界 Agent 客观状态
  ③ 社会/关系可行性                 → 查角色主观记忆
  ④ 叙事可行性                      → 查世界 Lore 因果链
  ⑤ 叙事轨道检查                    → 查叙事轨道 Agent
  ├─ 通过 → 继续
  └─ 不通过 → 叙事内拒绝（四种策略），流程终止
        ↓
[04 事件 Agent]
  生成事件（Tier 1 元数据 / Tier 2 摘要 / Tier 3 上下文 / Tier 4 原文）
  Tier 4 原文 = 呈现给玩家的叙事文本
  写入持久化存储 → 广播 Tier 1
        ↓ 广播（可并行）
  ┌─────────────────────────────┐
  ▼                             ▼
[世界 Agent]               [相关角色 Agent]
更新客观世界状态             各自生成主观版本
维护地点因果链               写入主观记忆
                             Tier A：完整解读
                             Tier B：简化接收
                             Tier C：不接收
        ↓
[08 Lore 固化]（异步）
  扫描事件原文，提取新陈述事实
  验证一致性 → 写入世界 Agent Lore 层 + NPC 档案
  维护因果链（当前状态 + 变更原因 + 历史状态）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[05 叙事轨道 Agent]（全程异步并行）
  持续读取事件流，自主判断是否偏离主线
  ├─ 一级：写入反思系统注入队列（内心声音提醒）
  ├─ 二级：写入 NPC 待注入队列（下轮对话信息渗透）
  └─ 三级：向事件 Agent 提交 NPC 自主行动请求
```

## 关键设计决策汇总

| 问题 | 决策 |
|------|------|
| 玩家输入形式 | 完全自由文本 |
| 拒绝的表达方式 | 全部叙事内化，不暴露系统语言 |
| 反思系统对玩家的可见性 | 声音可见，特质权重对玩家不可见 |
| 特质浮现方式 | 悄无声息，无仪式感 |
| 信号来源 | 双信号：语气（表达类特质）+ 选择（价值观类特质） |
| 主/客观记忆 | 世界 Agent 持有客观版本，角色 Agent 自行生成主观版本 |
| 主观记忆可否含错 | 可以，是设计意图 |
| NPC 层级 | 三级（A/B/C），支持动态升降级 |
| 世界演化方式 | 惰性求值，观测时推算 |
| 事件广播 | 按事件权重 × NPC 层级 × 关系距离过滤，支持二次传播 |
| 事件可撤销性 | 不可撤销 |
| Lore 存储方式 | 因果链（当前状态 + 变更历史） |
| 叙事轨道干预 | 三级递进，全程异步，LLM 自主判断 |
| 叙事轨道注入时机 | 方案 B：下一轮生效，信息层注入 |
| 初始化 | AI 无需人类输入，凭空生成世界 |
| 创世文档 | 持久化，支持多次游玩同一世界 |
| 错误处理 | 阻塞式，无降级，等待重试 |
| 语义判断归属 | 全部交给 LLM，架构只定义结构与流程 |
