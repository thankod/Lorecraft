# 01 输入层

## 概述

玩家以自由文本输入自己的**意图**，而非直接下达可执行指令。
输入层的职责是将自然语言意图转化为引擎可处理的结构化原子动作序列。

---

## 核心原则

**输入 = 意图，不是指令。**

玩家说"去找 boss 打败他"，这不是两条指令，而是一个意图。
引擎负责解释这个意图，判断其是否合理，以及如何在叙事中承接或拒绝它。

---

## 处理流程

### 第一步：语义解析

由 AI 对原始输入进行语义理解，提取：

- **核心意图**：玩家想达成什么目标
- **情绪色彩**：输入的语气、态度（喂给反思系统的信号 A）
- **涉及对象**：人物、地点、物品、关系
- **时间预期**：立即执行 / 计划 / 询问

### 第二步：原子动作拆解

将复合意图拆解为最小可执行单元的有序序列。

**示例：**
```
输入："去找 boss，打败他"

拆解结果：
  1. MOVE_TO(target=boss_location)       — 需要知道 boss 位置
  2. LOCATE(target=boss)                  — 需要 boss 对角色已知
  3. CONFRONT(target=boss, method=?)      — "打败"在无战斗游戏中语义待定
```

拆解后，每个原子动作独立进入后续检查流程。
序列中任意一步失败，不代表整体意图被拒绝，而是在失败点产生叙事响应。

### 第三步：语义消歧

部分动词在无战斗语境下语义模糊，需要在此阶段处理：

| 模糊词 | 可能含义 |
|--------|----------|
| 打败 | 说服、揭露、孤立、谈判胜出 |
| 攻击 | 言语冲突、施压、揭短 |
| 抢夺 | 偷取、索要、欺骗获取 |

消歧结果作为原子动作的参数传入仲裁层，而非由玩家显式指定。

---

## 与其他模块的接口

- → **反思系统**：输入的情绪色彩作为信号 A，更新特质权重
- → **仲裁层**：传入原子动作序列，逐步检查可行性
