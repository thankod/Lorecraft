# 00 初始化 Agent

## 概述

初始化 Agent 在游戏启动时运行一次，凭空生成完整的故事世界。
**无需任何人类输入。**

它是整个引擎的创世者——其输出作为"创世文档"分发给所有模块，
构成游戏世界的初始状态。每次启动都是一个独一无二的世界。

---

## 生成内容

初始化 Agent 一次性生成以下所有内容，确保内部逻辑自洽：

### 世界设定
```
背景与基调        时代、地点、社会结构、氛围
核心冲突         驱动整个故事的根本矛盾
隐藏的秘密       玩家需要逐步发现的真相
背后的阴谋       各方势力的真实意图
```

### 叙事结构
```
最终目标         故事的终点状态（不是唯一结局，而是可能抵达的终点）
起始事件         将玩家卷入故事的触发点
阶段大方向       故事分几个阶段，每阶段的叙事重心
关键叙事节点     各阶段中世界应当达到的状态（交给叙事轨道 Agent）
```

### 人物档案
```
Tier A NPC（3-7 人）
  完整背景、核心动机、深层动机（可能与表面动机不同）
  与其他角色的关系、持有的秘密、在故事中的作用

Tier B NPC（若干）
  基础背景、人格模板、在世界中的功能角色

玩家角色
  初始身份与处境（将玩家置于故事中心的起点设定）
  拥有的信息与缺失的信息
```

### 初始世界状态
```
各地点的起始状态
各势力的起始关系与资源
已经发生但玩家尚不知晓的背景事件
```

---

## 输出分发

生成完成后，创世文档分发至各模块：

```
初始化 Agent
    ├→ 世界 Agent Lore 层（Level 1）
    │     世界设定 + 背景事件 + 人物基础档案
    │
    ├→ 各 Tier A/B 角色 Agent
    │     各自的人物档案（身份、动机、关系、秘密）
    │
    ├→ 叙事轨道 Agent
    │     叙事结构（阶段划分、关键节点、最终目标）
    │
    └→ 事件 Agent
          起始事件（作为第一个事件注入，触发游戏开始）
```

---

## 生成的一致性保证

初始化 Agent 在单次调用中生成所有内容，而非分模块多次生成。
这确保了人物动机、世界秘密、叙事节点之间的内在逻辑一致——
所有内容出自同一个生成过程，不会产生模块间的矛盾。

生成完成后，各模块独立运行，不再回调初始化 Agent。

---

## 框架使用者的定制点

初始化 Agent 的系统提示（system prompt）是框架使用者的主要定制入口：

```
可定制：
  故事基调（黑色电影、政治惊悚、历史背景……）
  世界复杂度（人物数量、势力数量、秘密层数）
  叙事风格（极乐迪斯科式、古典侦探式……）
  玩家角色的初始定位
```

不定制则使用框架默认参数，生成完全随机的故事世界。
