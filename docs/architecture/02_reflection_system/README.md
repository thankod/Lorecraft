# 02 反思系统

## 概述

反思系统是引擎的**内层过滤机制**，以角色内心声音的形式存在。
它对游戏内的角色是透明的（角色不知道自己"有"这套系统），
但对游戏外的玩家可见——它是玩家行为模式的镜像投影。

---

## 工作时机

反思系统在仲裁层**之前**介入，作用于意图层面而非动作层面。

```
原子动作序列进入仲裁前
    ↓
反思系统评估意图
    ├── 合理 → 静默通过
    ├── 存疑 → 相关声音发言（隐式提醒）
    └── 明显不可行 → 声音主动拦截，提供叙事内反馈
```

反思系统不"拒绝"意图，它只是**让角色产生顾虑**。
玩家仍可坚持，但坚持本身会被记录为行为信号。

---

## 坚持机制

当玩家在声音提醒后仍然坚持相同意图：

1. **第一次坚持**：声音语气加重，提供更具体的警告
2. **确认坚持**（玩家明确表达"我就是要这么做"）：进入仲裁层，但附带"强行推进"标记
3. **仲裁通过**：执行，但事件 Agent 生成时附加负面后果权重
4. **仲裁不通过**：叙事内给出世界级别的阻断，不产生系统提示

坚持行为本身作为**信号 B**（选择信号）输入特质权重计算。

---

## 声音之间的辩论

当多个活跃特质对同一意图有不同立场时，触发简短辩论：

- 辩论在角色脑海中以内心独白形式呈现
- 每个声音只发言 1-2 句，保持节奏感
- 辩论结果不强制玩家选择，只呈现张力
- 两层声音可以参与同一场辩论（如"规划"反对，"戏谑"支持）
